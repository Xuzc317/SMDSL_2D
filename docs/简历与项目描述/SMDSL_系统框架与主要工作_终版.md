系统框架与主要工作（SMDSL 简历终版）
==================================

## 系统框架

项目采用四区声明式管线架构，各区之间通过结构化的中间表示（topology_bundle / RoboIR JSON / violation_report）解耦，单一职责且可独立测试。

第一区为空间感知引擎。输入为五种格式的 CAD 平面图（FloorplanQA JSON、工业 DWG/DXF、SVG 矢量图、PNG/JPG 栅格图、osmAG 拓扑文件），经格式派发器分流至对应解析器——JSON 走 FloorplanQA Schema 的结构化提取、DWG 经 LibreDWG 命令行工具解析后双轨提取几何墙线与图层语义、SVG 抽取 line/polyline/path 等基本图形元素、PNG 在 alpha 合成与自适应阈值后二值化——统一产出占据栅格。栅格随后经三步形态学后处理（双线墙焊接闭运算、外部连通域剔除、薄墙桥接）消除工业图纸的墙体缝隙与外部泛洪，再通过欧氏距离变换生成连续距离场，最后以带安全膨胀代价的 A* 搜索输出拓扑标签图与可通行热力图。该区全程禁止调用大模型，所有计算为确定性数学。

第二区为语义编译器。接收自然语言任务指令与第一区产出的局部环境切片（仅含邻近物体名称列表，不传全局地图），通过 DeepSeek API 将指令编译为声明式的 RoboIR 标准约束 JSON。编译过程中，System Prompt 嵌入多条硬性护栏——禁止大模型输出坐标/距离/角度数值、禁止使用时序命令词、强制输出 POMDP 风格的声明式约束表达式——并在编译完成后经枚举值校验与引用一致性检查拦截幻觉输出。编译失败时通过三级 JSON 容错解析（纯 JSON→剥离代码块→正则提取首个大括号对象）与指数回退重试机制（最多三次）保证链路鲁棒性。

第三区为物理约束求解器。接收第二区产出的 STL 约束列表与用户输入的运动轨迹，在第一区产出的距离场上通过双线性子像素插值逐轨迹点采样，计算 STL 鲁棒度 ρ = d_real − D_safe。ρ 大于零表示安全冗余，等于零表示临界，小于零判定为硬约束违反。越界点被统一处理为 d_real = 0，即越界等价于碰撞。求解器同步输出每个违规节点的精确世界坐标、碰撞深度与时间戳，为上层提供连续语义梯度而非简单的二值碰撞标记。

第四区为结构化反馈与闭环自愈。当第三区检测到约束违反时，自动根据违反类型映射到 FailureTaxonomy 九类错误枚举，提取最深违规点（最小 ρ 值对应的轨迹点）作为诊断证据，生成含错误分类、鲁棒度分值、违规坐标与建议修正方向的结构化诊断报告。闭环自愈测试脚本将该报告作为大模型重规划的上下文，大模型在执行最小责任层恢复（仅调整被违反的约束参数、不改变原始任务意图）后，系统基于新安全阈值调用 A* 重规划绕行路径，并在三轮内完成从碰撞违规到安全通过的完整修正。

## 主要工作

基础空间感知与拓扑提取。针对 FloorplanQA 等标准化户型基准数据，基于 NumPy 与 SciPy 编写图论与形态学处理管线。核心工作包括：实现 weld_double_line_walls 对工业 DWG 双线墙缝隙的障碍层闭运算焊接（核半径由物理墙厚与分辨率动态计算），实现 remove_exterior_freespace 基于四连通域分析与边缘接触判定的外部自由空间剔除（含安全护栏防止整图清零），实现 bridge_thin_walls 对门框细线分割房间的自由层闭运算合并，以及基于 scipy.ndimage.distance_transform_edt 的欧氏距离场计算和带安全膨胀代价（1/distance 惩罚项）的八邻域 A* 拓扑提取。在全量 1,981 个 FloorplanQA 布局上完成批量验证，单布局平均耗时 29.9 毫秒，零错误。

LLM API 接入与 Prompt 护栏设计。负责 DeepSeek API 的交互逻辑与容错机制设计。为抑制大模型在空间推理中的坐标幻觉，设计并迭代了 RoboIR 编译器的 System Prompt 护栏体系——核心约束包括禁止大模型自行计算任何坐标/距离/角度数值（数值只能来自用户原文或场景画像）、禁止输出命令式的"先...再..."时序表述、强制输出声明式 STL 约束表达式。编译后增设 JSON Schema 字段完整性校验与 STL 操作符枚举白名单检查。编写了三级 JSON 容错解析器（直接解析→剥离 Markdown 代码块→正则暴力提取首个大括号对象），配合三次指数回退重试将 API 调用成功率从原始约 88% 提升至接近全覆盖。

闭环验证流水线搭建。独立编写端到端闭环自愈测试脚本（约 970 行），串联"场景初始化→陷阱指令构造→合成穿墙轨迹→物理校验→结构化反馈生成→LLM 重规划→A* 绕行→再校验"的完整测试链路。实现当轨迹在距离场底图上发生物理违规时，自动提取最深违规节点（含时间戳、世界坐标、实际距离、鲁棒度偏差值 ρ），映射到 FailureTaxonomy 九类错误枚举（COLLISION/STL_VIOLATION/TIMEOUT 等）并生成结构化诊断报告，精准回灌给大模型触发最小责任层的约束修正与路径重规划。在受控二维测试场景中验证了最多三轮的自愈逻辑，ρ 值从初始的负值（碰撞）修正为正（安全通过）。
