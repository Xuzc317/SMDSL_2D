## Observed Schema Variations

1. **Top-level structure**: All DWG files produce JSON with at least `header` (or `HEADER`) and `OBJECTS` keys.
2. **Entity identification**: Entities are identified by `entity` string (e.g., LINE, CIRCLE, TEXT) or numeric `type` code. Both must be handled.
3. **Coordinate representation**: Points use `{x, y}` dicts within entity-specific keys (`start`/`end` for LINE, `center` for CIRCLE, `ins_pt` for TEXT/INSERT, `vertices` for POLYLINE).
4. **Coordinate systems**: INSUNITS varies across files. The dispatcher applies `_DWG_UNIT_TO_M` scaling; files without INSUNITS default to 1.0 (assumed meters).
5. **Layer references**: Layers are typically string names; in some cases may be numeric handle references (handled by dispatcher's `_extract_dwg_semantics`).
