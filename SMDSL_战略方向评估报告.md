## 一、核心发现

### 1.1 EDT + STL 组合在文献中完全空白（置信度：高）

经过 15+ 篇论文的对抗验证，**零篇已发表工作将 EDT/EDF 连续距离场与 STL 形式化约束结合在同一个路径规划框架中**。

文献中存在两个活跃但断裂的轨道：

| 轨道 | 代表工作 | 发布处 | 缺少 |
|------|---------|--------|------|
| **EDF/EDT 规划** | FS-Planner (RAS 2026), Safe Bubbles (ISRR 2024), ESDF 轨迹优化 (Sensors 2025) | RAS, ISRR | 时序逻辑、形式化验证 |
| **STL 规划** | MAPS² (IJRR 2025), STLCG++ (RA-L 2025), CBF+STL | IJRR, RA-L | 距离场、连续安全梯度 |

关键词检索 "Euclidean distance transform" + "signal temporal logic" 在 arxiv、Semantic Scholar、Google Scholar 上均返回零结果。

**→ 这意味着 SMDSL 在学术上是真正的新贡献，不是"又一个 planner"。**

### 1.2 "具身鸿沟"是这个领域最热的问题（置信度：高）

独立基准测试（SafeAgentBench, EARBench, SafeMindBench）一致确认：原始 LLM 产出的机器人安全计划仅有 **3.0-37.6%** 的安全率。这是量化的、可复现的严重缺陷，不是边缘案例。

| 模型 | DroneNav 安全率 | TabletopManip 安全率 |
|------|----------------|---------------------|
| GPT-4 | 37.6% | 30.8% |
| GPT-3.5 | 22.8% | 16.4% |
| Llama2-7b | 5.2% | 3.0% |

**SMDSL 的架构——LLM 输出声明式 STL 约束 + EDT 距离场处理连续碰撞检测——正是领域呼吁的混合方案。** SELP（Purdue, ICRA 2025）通过 LTL 约束编码达到 95.2% 安全率，证明了形式化方法是这个方向的最强防御类别。

### 1.3 连续场正在成为机器人安全的主流（置信度：高）

NVIDIA 的 nvblox（ICRA 2024）实现了 TSDF 177 倍加速、ESDF 31 倍加速（vs CPU voxblox），已集成到 Isaac ROS，被 Figure Robotics 等公司采用。SDF/ESDF 正在从学术工具转变为工业标准。

SMDSL 使用的 `scipy.ndimage.distance_transform_edt` 与 nvblox 在数学上同源——差异在于规模（研究原型 vs 生产系统）。

## 二、方向对比

### 方向 A：学术发表

**核心叙事**：EDT 连续距离场 + STL 形式化约束搜索 → 在窄通道和模糊场景中提供统计显著的安全裕度提升

| 维度 | 评估 | 依据 |
|------|------|------|
| **技术新颖性** | 🔴 极高 | EDT+STL 组合完全空白，25 claims 对抗验证无矛盾 |
| **发表可行性** | 🔴 高 | STL+planning 正在 IJRR/RA-L 发表；EDT-vs-costmap 是明确的 open question |
| **实验规模** | 🟡 中等 | 200 布局 × 5 对 + 3-5 baseline + 统计检验（配对 t-test / Mann-Whitney U） |
| **代码匹配度** | 🔴 高 | benchmark/ 框架已存在，EDT/A*/STL 管线完整，38 测试通过 |
| **主要差距** | — | 缺少真实硬件验证；缺少与 Nav2 costmap 的直接对比实验 |

**ICRA/IROS 发表公式**（已确认的模式）：
1. 新颖的安全机制（✅ 我们有）
2. 3-5 个竞争 baseline（⚠️ 需要补 Nav2 对比）
3. 仿真 + 真实硬件验证（⚠️ 缺硬件）
4. 开源代码（✅ GitHub 已配置）
5. 相比 SOTA 提升 10-20%（⚠️ 需系统实验）

### 方向 B：工业工具

**核心叙事**：可插拔的、ROS 兼容的、提供连续安全梯度的路径规划验证中间件

| 维度 | 评估 | 依据 |
|------|------|------|
| **技术新颖性** | 🟡 中 | Nav2 Collision Monitor 已有 Approach model（连续调速），但无 STL 形式化验证 |
| **落地可行性** | 🟡 中-低 | Autoware 提供了插件接口，但工业对 STL 验证的需求未被证明 |
| **工程工作量** | 🔴 极高 | 需要 ROS 2 集成、C++ 重写、实时性能（<10ms/query）、硬件部署 |
| **代码匹配度** | 🟡 中-低 | Python 2D 原型距离 C++ ROS 生产系统差距巨大 |
| **主要差距** | — | 无 ROS 2 集成、无 C++ 实现、仅 2D、无实时性能保证 |

**关键风险**：NVIDIA Isaac ROS 已经将 nvblox SDF 推入生产。如果我们要做"工业级连续安全场工具"，竞争对手是 NVIDIA 的工程师团队和 GPU 硬件优化——这不是一个 10K 行 Python 项目能正面竞争的。

## 三、性价比分析

```
                    投入          产出          性价比
方向 A（学术）      ████░░      ████████      ★★★★★  最优
方向 A+B（学术→工业）████████    ██████████    ★★★☆☆  理想但高风险
方向 B（纯工业）    ████████    ████░░░░      ★★☆☆☆  不推荐
```

### 推荐：方向 A 优先，方向 B 作为长期愿景

**理由**：

1. **低成本高回报**：现有的 benchmark 框架、38 测试、EDT/A*/STL 管线已覆盖学术论文所需的大部分基础设施。补 1-2 个月实验即可投稿。

2. **学术信用积累后，工业落地更容易**：一篇已发表的论文是工业推广的最强背书。MAPS² 在 IJRR 发表后，成为 STL 规划的事实标准引用。SMDSL 先发论文建立学术信用，再开源工具会更有说服力。

3. **方向 B 需要学术论文来证明价值**：当前没有人能用 STL 做形式化路径规划验证——这篇论文本身就是你区别于 Nav2/NVIDIA 的核心证据。没有论文之前，工业界不会采纳。

4. **两个方向不是对立而是顺序关系**：发论文 → 积累引用和可见度 → 吸引合作者/资金 → 工业级实现（C++、ROS 2、实时优化）。这是学术界→工业界最成熟的路径（nvblox 也是先从 ICRA 论文开始的）。

## 六、关键风险

1. **被 scoop 的风险**：EDT+STL 是空白地带，但其他人也可能发现。建议尽快完成实验、挂 preprint（arxiv）。

2. **NVIDIA nvblox 的挤压**：如果 nvblox 加入 STL 验证（可能性低但非零），SMDSL 的差异化优势会消失。窗口期可能只有 12-18 个月。

3. **实验效果不够显著**：如果 EDT vs costmap 的差异只有几个百分点，统计显著性可能不够。需要设计足够的窄通道场景来放大差异。

## 七、结论

**学术优先，工业跟进。**

现有代码资产与学术方向匹配度极高，与工业方向差距巨大。先发论文建立信用，再以论文为跳板做工业落地——这是 nvblox、FS-Planner、MAPS² 等成功案例的共同路径。

SMDSL 不需要在"学术"和"工业"之间二选一，而是需要在"先学术后工业"和"直接跳工业"之间做选择。调研结果强烈支持前者。

## 八、第三层调研补充发现（ROS 工具链层）

### 8.1 RTAMT：唯一的 STL+ROS 工具，但有结构性缺口

RTAMT（Runtime Temporal Logic Monitor）是目前**唯一**具有原生 ROS 集成、定量鲁棒性语义、支持过去和未来时序算子的 STL 运行时监控工具。但它的架构有一个根本限制：

> **RTAMT 的 STL 谓词操作在 ROS topic 数据流上（cmd_vel、odom、robot_pose），而不是空间安全表示（costmap、占据栅格、距离场）。**

这意味着任何空间安全谓词（如 `distance > 0.3m`）必须在 RTAMT 外部预先计算为标量信号，再传入 STL 监控器。

**→ SMDSL 的 EDT+STL 管线正好填补了这个"空间安全到时序逻辑的桥接"缺口。**

### 8.2 ROSMonitoring 2.0：有 STL 支持，但只有三元离散判定

ROSMonitoring 2.0 通过 Reelay oracle 支持 STL，但输出仅为三元离散判定（satisfied/violated/unknown），**不提供连续鲁棒度**。而 SMDSL 的 `FailureTaxonomy` 和 ρ 值正是连续鲁棒度的实现。

### 8.3 GP 距离场的量化证据

基于高斯过程的距离场方法在真实数据集上达到 **96-100% 无碰撞率**，对比 PRM（28-87%）和 A*（54-96%）。这为"连续距离场比二值/采样方法更安全"提供了独立的统计证据——虽然不是 EDT 的直接对比，但逻辑一致。

### 8.4 结构性的集成缺口（三层调研一致确认）

不存在任何工具同时做到：
1. 在工作空间上计算 EDT
2. 将 EDT 值用作 STL 谓词输入
3. 返回连续鲁棒度分数（而不仅仅是 pass/fail）

**这个三重缺口同时构成了学术贡献（新框架）和工业价值（填补工具链空白）。**
