# SMDSL (RoboIR) 项目 README 与全局工程总结 (封存版)

> ⚠️ 致后续接手的 Cursor Agent 或开发者：本 README 记录了 SMDSL (RoboIR) 项目从最初的理论狂想，到中间的工业级数据清洗阵痛，再到最终实现端到端闭环自愈的完整工程演进。请在基于此代码库继续开发时，严格遵守已确立的四大执行区解耦原则。

## 1. 核心初心与终极目标 (Mission & Vision)

**痛点：** 现有的视觉-语言-动作（VLA）模型（如 RT-2, OpenVLA）通常采用端到端直接输出离散Token的黑盒模式。当我们将海量的3D顶点矩阵或树状坐标拓扑丢给大模型时，它会陷入"空间盲区"，无法理解物理几何的关联性，导致在具有时序依赖的复杂任务中极易陷入局部最优解，且发生碰撞时完全不可追溯。

**解法：** 我们打造了 SMDSL (标准实现为 RoboIR) —— 这是一个介于 LLM 大脑与物理执行器之间的"中间表示层 (IR)"。不让大模型做数学题： 我们通过底层 API 库将空间计算（欧氏距离、最短路径、形态学闭包等）封装在底层。只让大模型讲逻辑： 强制大模型输出带有 Pose、Frame 等强类型声明，以及带有时空约束（STL）的声明式代码。从而在编译器阶段彻底绞杀"坐标系相加"等空间幻觉。

## 2. 三大 Demo 的演进血泪史：从实验室沙盒到工业级落地

### Demo 1：感知层 - 环境解析（从视觉玄学到图论确定性）

**抛弃深度学习：** 放弃依赖纹理的 U-Net，全面转向 100% 确定性的数学图论方案：欧氏距离变换 (EDT) + A* 漫水探测。

**真实工业 DWG 的毒打与重生：** 在接入 LibreDWG 提取真实的 .dwg 工业图纸后，我们成功克服了 320MB+ 内存崩溃（通过图层白名单过滤）、曲线丢失（通过参数方程离散化 SPLINE/ELLIPSE）以及最致命的"双线墙"漏水问题（通过物理尺度 0.3m 动态计算 Kernel 尺寸的形态学闭运算），最终实现了工业级的拓扑提取。

### Demo 2：认知层 - 语义编译（从黑盒到强类型约束）

**Spatial-RAG 落地：** 实现了空间检索增强生成 (Spatial-RAG) 机制，LLM 先从自然语言指令中提取实体名词，去拓扑图中进行局部采样，彻底切断了全局地图导致上下文崩溃的风险。

**安全护栏：** 强制大模型只做逻辑翻译，并在 UI 中打上"仅供人类阅读参考，机器人执行依据仍是 JSON"的醒目标签，捍卫系统边界。

### Demo 3：验证层 - 物理求解（从标量奖励到语义梯度）

**ρ(t) 鲁棒度曲线：** 底层 API 在距离场底图中进行双线性采样，严格计算出 STL（信号时序逻辑）的定量鲁棒度 $\rho = d_{real} - D_{safe}$。

**结构化失败反馈 (Structured Failure Feedback)：** 如果发生碰撞（$\rho < 0$），系统抛出包含 FailureTaxonomy.COLLISION 及具体违规坐标的结构化反馈，为大模型提供明确的"语义梯度"。

## 3. 终极闭环：端到端自愈测试 (Closed-Loop Recovery)

系统已在 `test_closed_loop_recovery.py` 中成功实现"故障驱动的自我优化"：接收故意诱导撞墙的陷阱指令。物理引擎精准捕获碰撞（如 $\rho = -0.2000$）并抛出结构化报错。LLM 根据报错执行最小责任层恢复 (minimum-responsible-layer recovery)，在不推翻原始意图的前提下，通过添加规避约束或 Waypoint，在 3 次重试内成功完成自纠错。

## 4. 项目代码边界与纪律

- `vlm_parser.py` (仅限语义逻辑)
- `astar_topology.py` (纯确定性图论)
- `spatial_api_stub.py` (纯代数与物理约束验证)

**严禁任何形式的跨界污染，大模型绝对不可直接参与底层数学计算。**
