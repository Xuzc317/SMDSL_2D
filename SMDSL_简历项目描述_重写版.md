## 简历正文

**面向具身智能的空间约束中间语言与验证系统 (SMDSL / RoboIR)**

**一句话概述**：从零独立设计并实现了一套面向机器人 2D/2.5D 避障规划的"空间约束编译-验证-自愈"闭环系统，将自然语言任务指令转化为可审计的物理安全约束，并在 1,981 个真实户型场景上以平均 29.9ms/场景完成拓扑提取，零失败。

---

**解决的问题**

现有视觉-语言-动作（VLA）模型采用端到端黑盒模式，直接输出动作指令——当机器人靠近墙体或障碍物时，系统无法解释"为什么这里不能走"，发生碰撞后也无法追溯根因。本项目提出一种声明式中间语言（RoboIR），核心理念是：**大模型只做逻辑翻译（"从哪里到哪里，避开什么"），底层数学计算（距离、碰撞、路径）全部交给确定性几何引擎——两者之间用 STL 约束语法作为"合同"**。类比：大模型写 SQL 的 WHERE 条件，底层引擎执行查询计划——谁该做什么，边界清晰。

**系统架构：四模块 Pipeline（~10,000 行 Python）**

```
CAD图纸(DWG/SVG/JSON/PNG/OSM)  ──→  [模块1] 空间感知引擎
                                            │  将建筑平面图转为连续距离场（每个像素到最近墙的精确距离）
                                            │  1981个布局测试, 零失败, 平均29.9ms/布局
                                            │
自然语言指令("从A移动到B,避开设备")  ──→  [模块2] 语义编译器
                                            │  LLM将指令编译为STL约束JSON(不计算任何坐标/距离)
                                            │  5条硬性护栏防止LLM产生空间幻觉
                                            │
                                            ──→  [模块3] 物理约束求解器
                                            │  在距离场上逐点验证轨迹: ρ = d_real − D_safe
                                            │  ρ<0 → 碰撞/违规; 输出违规点坐标(x,y)和碰撞深度
                                            │
                                            ──→  [模块4] 闭环自愈
                                                  将违规报告(含具体坐标)反馈给LLM
                                                  LLM修改约束 → A*重规划路径 → 再验证
                                                  3轮内完成自愈修正
```

**技术细节与工程挑战**

| 工程挑战 | 解决方案 | 实际效果 |
|:---|:---|:---|
| 工业 DWG 图纸中墙体用双线表示，栅格化后出现"墙缝"——A* 路径穿过墙体内部 | 对障碍层做动态核尺寸的形态学闭运算，核半径由物理墙厚/分辨率自动计算 | `historical_museum.dwg` 连通域从87个碎片→4个主区域 |
| PNG/SVG 格式图片的白色背景被识别为"自由空间"——路径规划泄漏到建筑外部 | 4-连通域分析→四边缘接触域判定为"外部"→覆写为障碍；加安全护栏防止内部清零 | PNG/SVG 路径下 A* 搜索严格限制在建筑内部 |
| A* 8邻域对角移动时，机器人可能从墙角"穿过" | 显式角切检测——对角移动时检查两个正交邻居的占据状态和距离场，任一不满足R_robot即阻止 | 零幽灵穿墙 |
| 6-DoF 机械臂轨迹传入时 Z 轴被静默忽略→假阴性碰撞(报告安全实则碰撞) | 已识别并记录为已知局限（文档中诚实标注）；后续版本计划加硬断言 | N/A（当前仅适用于2D移动底盘） |
| 距离场为初始化时的静态快照——不支持动态障碍物 | 标注为当前版本局限，下一版本计划实现增量EDT局部更新 | N/A |

**已知局限（诚实标注）**

- **2.5D 限制**：当前距离场仅覆盖 (x,y) 平面，Z 轴被忽略——适用于 AGV/移动底盘，不适用于 6-DoF 机械臂的空间避障
- **静态场景**：距离场为一次性快照，不支持动态障碍物（移动行人/其他机器人）
- **轨迹合成为线性插值**：当前未引入速度/加速度 profile（梯形速度优化已列入下一迭代计划）
- **LLM 修正为软约束**：LLM 重规划时可能改变原始任务意图——已设计 diff 硬校验方案待实现

## 版本二：精简版（适用于空间有限的简历）

**SMDSL — 面向具身智能的空间约束验证与闭环自愈系统** | 个人项目 | 2026.05 - 至今

独立设计并实现了一套将 CAD 建筑平面图转化为机器人可执行安全约束的完整管线（~10,000 行 Python，21 个模块）。

- **空间感知引擎**：支持 5 种 CAD 格式（DWG/SVG/JSON/PNG/OSM），工业 DWG 图纸经 7 层安全沙箱 + 形态学后处理（双线墙焊接/外部泛洪剔除/薄墙桥接）→ 连续 EDT 距离场 + 安全代价 A* 拓扑提取。1,981 个 FloorplanQA 布局测试零失败，平均 29.9ms/布局。
- **STL 约束编译与验证**：将自然语言指令编译为声明式 STL 约束（大模型不做数学），在距离场上通过双线性子像素采样逐点计算轨迹鲁棒度 ρ = d_real − D_safe；4 条验证轨迹定量对标（安全/贴墙/穿物/越界）。
- **LLM 闭环自愈**：违规时生成结构化诊断报告（含违规坐标、碰撞深度 ρ 值），反馈给 LLM 执行最小责任层恢复，A* 自动重规划——3 轮内完成修正。

技术栈：Python / Gradio / NumPy+SciPy / DeepSeek API / LibreDWG / OpenCV | GitHub: github.com/Xuzc317/SMDSL_demo
