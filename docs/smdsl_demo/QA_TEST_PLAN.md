# SMDSL（RoboIR）Gradio 调试面板 — 测试方案（v1）

本文档供**专职测试**执行回归与验收。被测对象为 `python -m smdsl_demo.app` 三 Tab 集成界面及与 `cad_parser`、`vlm_parser`、`spatial_api_stub`、`metrics` 的联动。

**范围外**：Rust 真编译器、生产部署、真实机器人硬件、第三方 LLM 计费与 SLA。

**通过准则**：无阻断级崩溃；核心路径可重复；结果与「四大执行区」一致（VLM 不做几何计算，几何由底层完成）。详见 `PROJECT_CONTEXT.md`。

---

## 1. 测试环境与前置条件

### 1.1 硬件 / 系统

- Windows 10/11（或团队约定 OS）
- 建议内存 ≥ 8GB；磁盘有足够空间写 `out/` 及缓存目录

### 1.2 软件

- Python 3.x（与仓库一致）
- 依赖：`gradio`、`openai`、`numpy`、`scipy`、`matplotlib`、`Pillow` 等（以能成功 `import smdsl_demo.app` 为准）

### 1.3 环境变量

| 变量 | 用途 | 测试组合 |
|------|------|----------|
| `DEEPSEEK_API_KEY` | Demo 2 编译、智能推断、摘要、Demo 3 诊断 | **有 key** / **无 key** |
| `NO_PROXY` | 本地 Gradio 若走系统代理易出现 502 | 建议：`localhost,127.0.0.1` |

**标准启动命令：**

```powershell
$env:NO_PROXY = "localhost,127.0.0.1"
# 测 Demo 2 / 诊断时再设（勿将密钥写入本文档或缺陷单）：
# $env:DEEPSEEK_API_KEY = "<从安全渠道获取>"
cd <REPO_ROOT>
python -m smdsl_demo.app
```

### 1.4 测试数据

- CAD 样本根：`data/cad_samples`（与 Gradio FileExplorer 根目录一致）
- 建议每类至少 1 个样本：
  - **JSON**：`data/cad_samples/floorplanqa/layouts/**/room_*.json`
  - **PNG**：透明底线稿、白底黑线各一
  - **SVG**：含 `<path>` 的导出（验证解析器覆盖）

## 3. Demo 1 — 环境感知（CAD / 拓扑）

### 3.1 功能用例

| ID | 场景 | 操作 | 期望结果 |

| D1-01 | JSON 主路径 | FileExplorer 或路径框选 `floorplanqa` 下 JSON → 解析 | 信息区含模式、栅格尺寸、可走比例、连通域/节点等；原图与拓扑相关图可加载 |

## 6. 端到端（E2E）场景（建议每版本 1 次）

### E2E-1：室内 JSON 闭环

1. Tab 1：`room_*.json` → 解析 → 选 A/B → 求路径  
2. Tab 2：写与场景相关指令 → 推断环境 → 编译  
3. Tab 3：加载 RoboIR、加载轨迹 → 求解 →（可选）诊断  

**通过**：全程无未捕获异常；Demo 3 反馈与轨迹可对照理解。

### E2E-2：PNG 弱语义

1. Tab 1：PNG 仅拓扑与路径  
2. Tab 2：手填 `nearest_objects` 再编译  
3. Tab 3：同上  

**通过**：与产品说明「PNG 无对象语义」一致。
